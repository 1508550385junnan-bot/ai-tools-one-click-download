# 需求锁定

## R1 API 功能完全重改
- 做什么：重做本地应用中的 API 配置功能，使 Claude Code、Hermes Agent 等支持项能够正确写入配置并给出清晰结果。
- 不做什么：不接入付费服务，不替用户申请或保存真实 API Key 到仓库，不扩大到未支持应用。
- 约束条件：保留现有桌面应用入口和 UI 流程；配置写入必须可测试、可回滚备份；失败原因必须明确返回。

## R2 修复 CC Switch 安装失败
- 做什么：修复 CC Switch 下载、静默安装、安装后检测失败的问题。
- 不做什么：不删除用户已有安装，不执行不可逆系统清理。
- 约束条件：优先使用官方 GitHub Release；安装后通过可靠路径或注册表检测；失败要记录可诊断日志。

## R3 检测已安装后支持更新
- 做什么：审查并完善“已安装版本检测后可更新”的逻辑，确保旧版本显示升级入口，安装流程可覆盖升级。
- 不做什么：不实现后台强制自动更新，不绕过用户点击安装/升级。
- 约束条件：兼容无法读取版本号的工具；版本比较不能只做字符串相等判断。

## R4 审查并修复同范围其他 bug
- 做什么：审查安装、检测、API 配置相关代码路径，修复会导致功能不可用、状态错误或明显误报的问题。
- 不做什么：不做视觉重设计，不做无关重构，不改 README 营销内容。
- 约束条件：所有修改需可验证；发现范围外问题仅记录。

## R5 打包并发布更新版本
- 做什么：将已修复应用升级到新版本，构建 Windows 单文件 exe，并同步到 `1508550385junnan-bot/ai-tools-one-click-download`。
- 不做什么：不强制覆盖远端历史，不提交本地下载缓存、构建临时目录或敏感配置。
- 约束条件：版本号、`version.json`、应用内显示和发布产物名称必须一致；打包前后运行测试和基础安全检查。

## R6 修复 Hermes 已安装但显示未下载
- 做什么：检测本机已通过 pip 安装的 Hermes Agent，即使当前进程 PATH 未刷新，也能通过常见 Scripts 路径识别。
- 不做什么：不改变 Hermes 的 API 配置流程。
- 约束条件：必须保留 `hermes version` 子命令验证；检测失败时给出真实失败，不静默伪装成功。

## R7 添加 OpenClaw 下载/安装功能
- 做什么：在 AI 编程工具列表中新增 OpenClaw，通过官方推荐的 npm 全局安装方式安装，并能检测安装状态。
- 不做什么：不做 OpenClaw API 配置页，不改其他工具安装流程。
- 约束条件：安装命令、包名和可执行命令必须来自官方站点或 npm 注册信息。

## R8 在线检测 AI 工具新版本并提示升级
- 做什么：启动主界面后联网查询 npm/PyPI/GitHub 最新版本，已安装工具低于远端版本时显示“升级”。
- 不做什么：不阻塞启动，不因网络失败影响本地安装检测。
- 约束条件：网络失败必须回退到静态版本或保持现状；版本比较使用统一版本比较逻辑。

## R9 验证低版本远程升级提示
- 做什么：测试当前应用在远端 `version.json` 版本高于本地时会提示更新，远端版本相同或更低时不提示。
- 不做什么：不自动覆盖安装 EXE；只验证检测与下载入口逻辑。
- 约束条件：低版本升级判断必须使用数字版本比较，而不是简单字符串不相等。
